# Инициализирующий промпт для Codex

Скопируй этот текст в новый поток Codex в корне репозитория.

---

Ты работаешь над дипломным проектом «Автоматизированная подсистема моделирования и интерпретации сетей Петри». Это C++-ядро для общей системы: вторая часть проекта уже занимается визуализацией и передаёт модель сети Петри в JSON. Нужно реализовать вычислительную часть, которая принимает JSON, моделирует сеть Петри, запускает DFS/BFS/Dijkstra и возвращает JSON-результаты для Python/визуализации.

Сначала прочитай файлы `AGENTS.md`, `PROJECT.md`, `PLAN.md`, `API_CONTRACT.md`, `README.md`. Следуй им как источнику требований.

Начни с минимального вертикального среза:

1. Создай C++20/CMake каркас проекта.
2. Подключи JSON-библиотеку `nlohmann/json` и один тестовый фреймворк (`doctest` или `Catch2`).
3. Реализуй модуль `core_pn`:
   - `Place`, `Transition`, `Arc`, `Marking`, `PetriNet`;
   - загрузка из JSON;
   - валидация входной модели;
   - определение разрешённых переходов;
   - атомарное срабатывание перехода.
4. Реализуй `Interpreter`:
   - `step()`;
   - `run(max_steps, strategy=first_enabled)`;
   - журнал `SimulationEvent`.
5. Реализуй модуль `graph_models`:
   - простой directed graph;
   - adjacency list;
   - минимальную FSF-структуру;
   - построение ограниченного `reachability_graph` из сети Петри.
6. Реализуй алгоритмы `dfs`, `bfs`, `dijkstra` через единый интерфейс `run(graph, params) -> AlgorithmResult`.
7. Реализуй JSON service adapter:
   - `simulate_request(json) -> json`;
   - `algorithm_request(json) -> json`.
8. Добавь тесты:
   - загрузка `examples/mutex.json`;
   - валидация некорректных дуг;
   - проверка разрешённых переходов;
   - срабатывание перехода изменяет токены;
   - BFS на reachability graph находит путь к целевой маркировке;
   - Dijkstra использует `fire_time` как стоимость.
9. Обнови README командами сборки и запуска.

Ограничения:

- Не реализуй цветные/ингибиторные/стохастические сети в первой версии.
- Не добавляй HTTP-сервер, пока не готова библиотечная часть.
- Не меняй JSON-контракт без обновления `API_CONTRACT.md`.
- Возвращай структурированные ошибки, а не только строки.
- После изменений запусти сборку и тесты:

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug
cmake --build build
ctest --test-dir build --output-on-failure
```

В конце дай краткий отчёт: что реализовано, какие файлы изменены, какие тесты прошли, что делать следующим шагом.
